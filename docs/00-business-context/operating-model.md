# Operating model

JBB runs on eleven user roles. The role boundaries that matter most — because the business needs them clean — are:

- **Site Manager vs Foreman / Site Team.** Site Manager owns live-site delivery (sequencing, quality checks, snag coordination, RAMS enforcement, escalation, on-site H&S confirmation). Foreman / Site Team owns the workface — daily progress input, photos, attendance, housekeeping, immediate issue reporting up to the Site Manager.
- **H&SO vs OCC.** H&SO owns the H&S framework, audit regime, inspection templates, incident governance, corrective action oversight, mobilisation H&S, coaching. OCC owns the admin: subcontractor onboarding records, insurance / cert / CIS / RAMS collection, expiry tracking, filing, register maintenance. OCC does not own live-site safety; H&SO does not own office admin.
- **PM vs QS/Estimator.** PM owns programme, drawings, client liaison, coordination, the paperwork side of H&S. QS/Estimator owns tender build-up, package comparisons, award support, valuations, variations pricing and final account.
- **Architect vs Client.** Architect/Designer/Consultant is the technical author and reviewer. Client / Homeowner is the end customer who selects, approves, instructs, pays and signs off.

Each lifecycle workflow names exactly which role owns it, which roles approve, and which contribute. See [`/05-data-model/permissions-matrix.md`](../05-data-model/permissions-matrix.md).
