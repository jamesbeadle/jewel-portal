# P02 — Finance Director (FD)

**Type:** Internal executive
**Reports to:** Directors / MD
**Frequency on platform:** Daily — heavy on the cashflow dashboard
**Status:** Draft

## Goals — the platform's primary driver
- Hold an accurate cashflow forecast across the project portfolio without rebuilding it weekly in Excel.
- Produce the zero-rated VAT analysis at project completion from project data rather than from scratch.
- Hand JPMS-published project data cleanly into the accountancy systems they run downstream (Xero, Brightpay, Dext, Chaser HQ).

## Pain points (current state)
- Cashflow forecast accuracy depends on knowing real completion %, which is unreliable today.
- Cashflow rebuilt weekly in Excel from scattered sources; cross-entity charges manual.
- VAT analysis at project end built from scratch in Excel.

## Involvement across workflows
- **Owner on:** 07 (Valuations, Cashflow & Forecasting — the FD's primary surface) and 08 (Project Close-Out — settlement and VAT slice).
- **Approver on:** 03 (above-threshold subcontractor awards), 05 (above-threshold variations), 07 (payment-release decisions), 08 (retention release, final VAT outcome).
- **Read on:** every project so the forecast and settlement are well-informed.

## Permissions (coarse)
- Owner on the cashflow forecast and the project settlement / VAT analysis.
- Approver on cost-code overrun, valuation release at threshold, and retention release.
- Read across all projects.

## Devices
- Desktop primary. Mobile for approvals.

## Notes
- The FD's day job — AP, AR, payroll, inbox triage, statement reconciliation — runs in Xero / Dext / Brightpay / Chaser HQ. That work is **not** a JPMS workflow. JPMS publishes project data (work orders, valuations, approved timesheets, settlement records) that those tools consume.
