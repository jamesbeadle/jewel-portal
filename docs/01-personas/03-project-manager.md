# P03 — Project Manager

**Type:** Internal
**Reports to:** Directors / MD
**Frequency on platform:** Daily
**Status:** Draft

## Goals
- Coordinate the project end-to-end with one source of truth, not five.
- Keep the programme reflecting site reality without re-keying.
- Run client / consultant correspondence and approvals through the system so nothing gets buried in inboxes.
- Manage RFIs, submittals, variations and NoDs as one connected change layer.

## Pain points (current state)
- Programme is rebuilt manually from MS Project + Excel; reconciliation with the valuation is manual.
- Three separate Excel logs for variations, RFIs and delay notices.
- Drawing supersedure relies on memory.
- Approvals scattered across emails.

## Involvement across workflows
- **Owner on:** 01 (Drawing Receipt & Document Control), 05 (RFIs, Submittals, Variations & Delays), 06 (Site Delivery, Programme & Reporting — project-management slice; Site Manager owns the live-site slice).
- **Contributor / approver on:** 00 (handoff from sales / qualified opportunity), 02 (tender — works with QS), 03 (procurement award), 07 (triggers valuations + cost-code overrun routing), 08 (commercial sign-off on close-out), 09 (project-level reporting feeds portfolio).
- **Read on:** every workflow on their assigned projects.

## Permissions (coarse)
- Owner on project records, programme, drawing register, project change register, correspondence log.
- Approver on drawing supersedure overrides, RFI sign-off, NoD issue, site report sign-off.
- Read across all assigned projects.

## Devices
- Laptop primarily; mobile for approvals; tablet on site.

## Notes
- PM is the **paperwork** side of H&S — RAMS issue, induction admin support, audit follow-up. The H&SO owns the framework and assurance.
