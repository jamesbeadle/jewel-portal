# 07 — Valuations, Cashflow & Forecasting

**Lifecycle stage 07.** Detailed workflow: [`/03-workflows/07-valuations-cashflow-forecasting.md`](../03-workflows/07-valuations-cashflow-forecasting.md).

## Purpose

The commercial outputs. JPMS produces **three** primary commercial outputs from project data, each for a different audience:

- **Programme Valuation Report (PVR)** — the per-Claim-Period valuation issued to the client / architect / CA.
- **CVR (Cost-Value Reconciliation)** — internal commercial control for the QS and PM: actual vs forecast vs tender per package, margin per trade, Prelims and EOTs visible separately, variations against the original BoQ headings and the central register. **Replaces JBB's Excel CVR workbook and removes any need for Planyard.**
- **Cashflow forecast** — Director / FD view of forward income, commitments and predicted completion across the portfolio.

All three share one source of truth so they can never disagree. Timesheet capture for cost-code allocation feeds the CVR's Cost Incurred and the cashflow's labour commitment, and powers completion-% prediction.

## Key concepts at this stage

Valuation, Claim Period, Programme Valuation Report, CVR Snapshot, Forecast Component, Margin Trace, QS Accrual, Prelim Item, Prelim Forecast Entry, EOT (Extension of Time), Daywork, Contra Charge, Subcontractor Retention, Cashflow Forecast Snapshot, Timesheet, Timesheet Approval, Cost Code Budget, Cost Code Allocation.
